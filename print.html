GitHub Pages でmdbook を使う

mdbook はGitHub, GitHub Actions を使うことでGitHub Pages でドキュメントを公開できます。個人の技術ブログに使うこともできそうだと感じています。

ここでは、どのように設定すれば利用できるかの例を記します。

前提

  • GitHub アカウントを持っている。
  • ローカルで作った原稿をcommit する

    • ローカルにmdbook がインストールされている(cargo も)

      とはいえ、cargo install mdbookをするだけだし、cargo はrust をインストールすれば入る

    • mdbookで初期化したディレクトリを用意してある

  • ディレクトリ構成は
  ディレクトリ構成
  │  book.toml
  │  
  ├─.github ← あとで解説します
  │  ├─actions
  │  │  └─build
  │  │          Dockerfile
  │  │          
  │  └─workflows
  │          main.yml
  │          
  ├─book
  │          
  └─src
      │  原稿.md
      └─ SUMMARY.md

全体的な流れ

  1. GitHub でリポジトリを作る
  2. GitHub Actions 用の設定ファイルを作る
  3. デプロイキーの設定
  4. 原稿をアップロードする

GitHubでリポジトリを作る

pages を使いたいのだからリポジトリを作るのは当たり前だろ！！！と思うかもしれませんが2つほどリポジトリの構成を考えられるのでその説明をします。それは

原稿をアップロードするリポジトリとpages のリポジトリを

  1. 別にする
  2. 一緒にする

です。

別にするメリット

✓ 下書きも含め原稿を公開せずに済む

✓ すでに公開しているpages に組み込める

一緒にするメリット

✓ ちょっとだけ設定が楽

✓ 他の人と協力して原稿が作成できる

GitHub Actions 用の設定ファイルを作る

  ├─.github
  │  ├─actions
  │  │  └─build
  │  │          Dockerfile
  │  │          
  │  └─workflows
  │          main.yml

これの解説です。 このディレクトリ構成でDockerfile とmain.yml を作ればいいです。

Dockerfile

GitHub Actions で動かすDockerの設定です

FROM rust:latest

RUN cargo install mdbook --no-default-features --features output --vers "^0.3.5"

CMD ["mdbook", "build"]

main.yml

main ブランチのsrcディレクトリにある.mdファイルとbook.tomlが変更されたときにactions が動くように設定されています。

  • リポジトリ別の場合
name: CI/CD
on:
  push:
    branches:
    - main
    paths:
    - "src/**.md"
    - "book.toml"

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Build
        uses: ./.github/actions/build
      
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        env:
          ACTIONS_DEPLOY_KEY: ${{ secrets.鍵の名前 }}
          external_repository: ユーザ名/リポジトリ名
          PUBLISH_BRANCH: pages で使うブランチ名
          PUBLISH_DIR: ./book
  • リポジトリ一緒の場合
name: CI/CD
on:
  push:
    branches:
    - main
    paths:
    - "src/**.md"
    - "book.toml"

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Build
        uses: ./.github/actions/build
      
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        env:
          ACTIONS_DEPLOY_KEY: ${{ secrets.鍵の名前 }}
          PUBLISH_BRANCH: pages で使うブランチ名
          PUBLISH_DIR: ./book

ほとんど同じですが、リポジトリ別の場合はexternal_repository を追加して上げる必要があります。

デプロイキー

実は上記の方法のどちらを選択したかによってデプロイキーの設定が少し変わります。

  • 別のリポジトリの場合

    • ローカルにおいてssh-keygenでSSH鍵を作ります。
    ssh-keygen -t rsa -b 4096 -f gh-pages # これだとgh-pagesという名前でSSH鍵ができる

    • 公開鍵(gh-pages.pub)をPages用リポジトリのSettings > Deploy keysへ登録

      • Allow write accessにチェック✓

    • 秘密鍵を原稿を上げるリポジトリのSettings > Secretsへ登録

    • このとき設定した秘密鍵の名前がmain.ymlのACTIONS_DEPLOY_KEYで使われます。

    • 違う名前で登録した場合どうなるかわからないので、同じ名前にすると良いと思います。

  • 同じリポジトリの場合

    • アカウントのsettings > Developer settings > Personal access tokensでトークンを発行

      • repoにチェック✓

    • リポジトリのSettings > Secretsへ発行したトークンを登録

      • このとき設定した秘密鍵の名前がmain.ymlのACTIONS_DEPLOY_KEYで使われます。

原稿をアップロードする

アップロードするだけです。

すると自動でpages のリポジトリにbuild してくれます。

powershellでディレクトリ構成図

これだけ！

tree [ターゲットのディレクトリ] /f | Out-File -Encoding utf8 [テキストファイルの出力先/名前.txt]

実行例

tree C:\ /f | Out-File -Encoding utf8 C:\out.txt

(現実的な)実行例

めちゃくちゃ長くなるので上記を実際に実行するのはおすすめしません。

以下を試してみてください。

# デスクトップへ移動
$DESKTOP = [Environment]::GetFolderPath("Desktop") # デスクトップのパス
cd $DESKTOP

# テスト用のフォルダとファイルを作る
mkdir ./treetest; mkdir ./treetest/test
cd treetest
New-Item -type file ./test1.txt; New-Item -type file ./test/test2.txt
tree $DESKTOP/treetest /f | Out-File -Encoding utf8 $DESKTOP/treeout.txt
cd $DESKTOP

# テストのディレクトリを消す
Remove-Item $DESKTOP/treetest -Recurse

# 図を出す
start treeout.txt

treeout.txt を消すには、デスクトップにあるこのファイルを手で消すか、続けて

# treeout.txtを消す
Remove-Item $DESKTOP/treeout.txt

を実行してください。
